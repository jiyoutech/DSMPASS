# 变更日志

项目在 pre-1.0 阶段尽量遵循语义化版本。小版本仍可能包含行为变化，升级前请阅读对应版本说明。

## 0.8.27

- 安装完成后延迟同步 DSM 套件 `INFO` 的 `adminport`，处理 DSM 在生命周期脚本之后再次覆盖入口端口的情况。
- 当管理端口不是 DSM 套件中心固定入口端口时，后端会在固定入口端口提供 HTTPS 跳转到真实管理端口，避免套件中心打开按钮失效。

## 0.8.26

- 修复 DSM 安装流程在 `postinst` 后重新覆盖套件 `INFO` 时，管理入口端口又回落到 25000 的问题。
- `start` 和 `status` 生命周期会按 `dsmpass.env` 的真实监听端口同步 DSM 套件中心入口端口。

## 0.8.25

- 修复 0.8.24 安装阶段同步 DSM 套件入口端口时可能导致 `postinst` 失败的问题。
- 套件启动时会再次按 `DSMPASS_GO_LISTEN` 同步 DSM 套件中心入口端口，避免入口端口回落到 25000。

## 0.8.24

- 修复 SPK 安装向导里修改管理端口后，后台监听端口没有同步更新的问题。
- 安装后会同步更新 DSM 已安装套件 `INFO` 里的 `adminport`，避免 DSM 套件中心入口继续指向 25000。

## 0.8.23

- 管理后台侧边栏品牌图标改为复用新版 `favicon.svg`，避免继续显示旧的内联 SVG 图标。
- 调整品牌图标样式，保留新版多色图标，不再强制套用旧的蓝底白色描边。

## 0.8.22

- `external-accounts` 和 `provider-groups` 管理接口改为 Go 侧统一计算短 `subject_hash`，避免 SQLite 环境缺少 hash 函数时泄露原始身份源 ID。
- 管理端列表接口补充分页、搜索和筛选能力，降低大规模同步数据下的页面和接口压力。
- 增加同步日志和登录审计日志保留清理，避免日志长期增长占用数据库空间。

## 0.8.21

- 浏览器直登模式下强制校验 DSM 地址和 DSM Auth API 协议必须与 IDP 协议一致，避免浏览器侧登录流程因跨协议访问失败。
- 系统设置页在 DSM 登录配置中增加协议一致性提示，并在保存前拦截不一致配置。

## 0.8.20

- 证书上传接口返回结构化证书信息，包括 Subject、Issuer、有效期、自签状态和证书类型。
- 证书上传成功提示会明确展示“测试证书 / 自签证书 / 正式证书”，方便管理员确认验证结果。

## 0.8.19

- 移除管理端重启 API 和前端按钮，避免后台触发套件进程重启。
- 移除未使用的 Relay journal/recover 管理接口，减少系统级危险操作入口。

## 0.8.18

- 认证端证书上传后会自动读取证书里的 DNS 域名，并同步更新认证入口域名和 IDP 地址。
- 管理端不再提供证书上传入口，继续使用 DSMPASS 自签证书。
- 管理端自签证书主体信息改为 DSMPASS。

## 0.8.17

- 移除复杂防火墙/安全组模块和独立防火墙日志库。
- IDP 入口会校验访问 Host，非系统配置里的 IDP 地址会直接拒绝。
- 管理端只保留「仅允许内网访问」开关，关闭时不按来源网段限制。

## 0.8.16

- 系统设置拆分为基础配置、防火墙、DSM 登录、证书与路由、后台账号子菜单。
- 防火墙页改为类似 ECS 安全组的基础入站规则体验，管理端口和认证端口分别配置独立有序规则和默认策略。
- 管理端口默认禁止未匹配来源并放行本机和内网；认证端口默认放行所有来源。
- 防火墙拦截日志写入独立 `firewall.db`，不写入主业务库；防火墙页分开展示拦截日志和认证登录日志。

## 0.8.15

- 系统设置页重构为入口地址、访问控制、DSM 登录、证书与路由等生产化配置区块。
- 访问控制增加「仅内网访问」快捷按钮，并在保存管理端网段时校验当前访问 IP，避免误配置后无法进入后台。
- 增加管理端重启按钮，用于管理端证书或套件侧配置变更后从后台触发套件重启。

## 0.8.14

- 移除用户、同步日志和登录审计统计条里的筛选结果统计项。
- 系统设置增加管理端口和认证端口访问网段控制；管理端默认仅允许本机和内网访问，认证端默认允许所有网络访问。
- 支持分别上传管理端口证书和认证端口证书，认证端证书上传后可在后台重启认证路由生效。

## 0.8.13

- 冲突处理弹窗改为同类冲突批量保存，不再逐条保存。
- 批量保存命名冲突后会自动尝试开通 DSM 用户或部门；保存失败时保留冲突输入，管理员可继续修改。
- 冲突处理改为卡片式逐组处理，当前组完成后再显示下一组。

## 0.8.12

- 修复 GitHub Actions 打包 SPK 时 Go build cache 路径不兼容 Linux runner 的问题。
- 放大冲突处理弹窗，增加冲突表格横向空间和弹窗内容可视高度。
- 调整前端构建体积提示阈值，减少单页后台应用在 CI 中产生误导性 warning。

## 0.8.11

- 重新发布 0.8.10 的同名用户处理改进，触发 GitHub Release 重新生成 SPK 附件。

## 0.8.10

- 用户冲突明确区分「飞书内重名」、其他身份占用和旧版 DSM 同名冲突记录，避免管理员误判冲突来源。
- 冲突处理窗口会把飞书内重名用户按飞书姓名分组放在一起展示，方便对照邮箱、手机号、身份 ID 和部门后改名。
- DSM 本地已有同名用户且飞书侧用户名唯一时，直接建立飞书身份到 DSM 用户的映射并标记为「已关联」；已有映射不会在后续同步中因飞书同名再次变成冲突。

## 0.8.9

- 打开身份源详情时，如果仍有同名部门或同名用户冲突，会自动弹出冲突处理窗口。
- 冲突处理窗口集中展示部门和用户冲突，支持在同一处对照飞书信息后修改最终 DSM 名称。
- 用户冲突信息补充飞书邮箱和脱敏手机号，普通用户列表里的冲突行也会高亮显示。
- 卸载套件时会尝试删除 Helper sudo 规则，并在卸载向导和文档中提示 SSH 确认及手动删除命令。

## 0.8.8

- 同名飞书部门和同名飞书用户不再自动决定最终 DSM 名称；系统先生成临时名并标记为冲突，管理员可在后台参考飞书信息手动修改。
- 部门名冲突处理完成前，同步会阻断 DSM 用户和成员关系开通，避免权限落到错误部门。
- 安全检查和身份源详情页增加用户冲突提示。

## 0.8.7

- 增加 GitHub Release 自动打包流程，推送 `v*` 标签时自动构建 SPK 并上传 Release 附件。
- README 补充 SPK 安装、首次网页初始化和飞书配置主流程。

## 0.8.6

- 飞书同名部门同步到 DSM 时改用完整部门路径生成部门组名，例如 `matrix/sup1/sup2/sup5` 会生成 `matrix_sup1_sup2_sup5`；不重名部门继续使用原部门名。

## 0.8.5

- 修复飞书多部门用户同步：优先读取用户对象里的 `department_ids`，一个用户属于多个部门时会建立多条成员关系。
- 部门成员接口补充分页读取，避免只同步第一页导致部分用户显示无部门。
- 已存在的部门成员关系在重新同步命中时会恢复为 active，避免历史漏同步状态影响后续结果。
- 管理后台将飞书通讯录关系文案从「群组」调整为「部门」。

## 0.8.4

- 身份源详情页的操作按钮改为独立 loading：同步、暂停/恢复登录、暂停/恢复同步、清理、删除、保存、允许/禁止登录互不影响。
- 同步完成后的数据刷新改为静默刷新，避免点击「同步」时表格或其他按钮一起进入 loading。
- 用户、群组、同步日志、登录审计里的「刷新」按钮增加独立 loading。

## 0.8.3

- 合并 Helper 权限提示中的「重启 Helper」和「重新检查」按钮，点击后会先重启 Helper，再自动检查权限。
- Helper 未完成 SSH 提权时，点击按钮直接提示先执行提权脚本。

## 0.8.2

- 修复点击「重新检查」时 Helper 权限提示区域短暂卸载、闪动或布局跳动的问题。

## 0.8.1

- Helper 权限异常提示里的「重新检查」改为只调用状态接口，不触发页面刷新。
- 重新检查完成后会明确提示检查通过、Helper 不可连接或 DSM 用户/群组命令权限异常。
- 身份源页不再常驻显示 Helper 权限提示；点击「新建」时弹窗说明必须先修复 Helper 权限。
- Helper socket 检查步骤补充 `ls` 的用途和正确输出示例。

## 0.8.0

- 增加管理后台「重启 Helper」能力，完成 SSH 提权脚本后无需重启整个套件。
- Helper 权限异常时禁止新建身份源，避免未提权状态下进入同步和 DSM 账号开通流程。
- SPK 内新增 Helper 管理脚本，并在 sudo 规则中允许套件用户调用该脚本。
- 更新 Helper 权限排障文档，提权脚本执行后改为在 UI 中重启 Helper。

## 0.5.0

- IDP 协议和 IDP 端口变化时，独立重启 IDP listener。
- IDP 路由变化时，管理后台 listener 保持运行。
- 持久化 IDP 地址协议变化，保存后和重启后身份源 Launch URL、Callback URL 都使用所选协议。
- 明确管理后台里的字段含义：开关控制的是 IDP 协议，不是管理后台协议。
- 保留 DSM 默认端口行为：HTTP 使用 `5000`，HTTPS 使用 `5001`，自定义 DSM 端口会保留。
- 用户可配置端口限制在 `1025-65535`。

## 0.4.0

- 增加 SPK 安装时的管理后台端口配置。
- 拆分管理后台和 IDP 路由概念。
- 增加 IDP 协议和 IDP 端口校验。
- 配置端口被占用时不再自动 fallback 到其他端口。
- 增加 DSM 默认端口派生：HTTP 对应 `5000`，HTTPS 对应 `5001`。

## 0.3.0

- 增加 DSM 7 SPK 早期打包流程。
- 增加访问主机、对外 URL、DSM 跳转地址和 DSM Login API 的运行时系统设置。
- 改进首次启动配置流程。

## 0.1.0

- 初始预发布打包基线。
