# 变更日志

项目在 pre-1.0 阶段尽量遵循语义化版本。小版本仍可能包含行为变化，升级前请阅读对应版本说明。

## 0.8.6

- 飞书同名部门同步到 DSM 时改用完整部门路径生成部门组名，例如 `matrix/sup1/sup2/sup5` 会生成 `matrix_sup1_sup2_sup5`；不重名部门继续使用原部门名。

## 0.8.5

- 修复飞书多部门用户同步：优先读取用户对象里的 `department_ids`，一个用户属于多个部门时会建立多条成员关系。
- 部门成员接口补充分页读取，避免只同步第一页导致部分用户显示无部门。
- 已存在的部门成员关系在重新同步命中时会恢复为 active，避免历史漏同步状态影响后续结果。
- 管理后台将飞书通讯录关系文案从「群组」调整为「部门」。

## 0.8.4

- 身份源详情页的操作按钮改为独立 loading：同步、暂停/恢复登录、暂停/恢复同步、清理、删除、保存、允许/禁止登录互不影响。
- 同步完成后的数据刷新改为静默刷新，避免点击「同步」时表格或其他按钮一起进入 loading。
- 用户、群组、同步日志、登录审计里的「刷新」按钮增加独立 loading。

## 0.8.3

- 合并 Helper 权限提示中的「重启 Helper」和「重新检查」按钮，点击后会先重启 Helper，再自动检查权限。
- Helper 未完成 SSH 提权时，点击按钮直接提示先执行提权脚本。

## 0.8.2

- 修复点击「重新检查」时 Helper 权限提示区域短暂卸载、闪动或布局跳动的问题。

## 0.8.1

- Helper 权限异常提示里的「重新检查」改为只调用状态接口，不触发页面刷新。
- 重新检查完成后会明确提示检查通过、Helper 不可连接或 DSM 用户/群组命令权限异常。
- 身份源页不再常驻显示 Helper 权限提示；点击「新建」时弹窗说明必须先修复 Helper 权限。
- Helper socket 检查步骤补充 `ls` 的用途和正确输出示例。

## 0.8.0

- 增加管理后台「重启 Helper」能力，完成 SSH 提权脚本后无需重启整个套件。
- Helper 权限异常时禁止新建身份源，避免未提权状态下进入同步和 DSM 账号开通流程。
- SPK 内新增 Helper 管理脚本，并在 sudo 规则中允许套件用户调用该脚本。
- 更新 Helper 权限排障文档，提权脚本执行后改为在 UI 中重启 Helper。

## 0.5.0

- IDP 协议和 IDP 端口变化时，独立重启 IDP listener。
- IDP 路由变化时，管理后台 listener 保持运行。
- 持久化 IDP 地址协议变化，保存后和重启后身份源 Launch URL、Callback URL 都使用所选协议。
- 明确管理后台里的字段含义：开关控制的是 IDP 协议，不是管理后台协议。
- 保留 DSM 默认端口行为：HTTP 使用 `5000`，HTTPS 使用 `5001`，自定义 DSM 端口会保留。
- 用户可配置端口限制在 `1025-65535`。

## 0.4.0

- 增加 SPK 安装时的管理后台端口配置。
- 拆分管理后台和 IDP 路由概念。
- 增加 IDP 协议和 IDP 端口校验。
- 配置端口被占用时不再自动 fallback 到其他端口。
- 增加 DSM 默认端口派生：HTTP 对应 `5000`，HTTPS 对应 `5001`。

## 0.3.0

- 增加 DSM 7 SPK 早期打包流程。
- 增加访问主机、对外 URL、DSM 跳转地址和 DSM Login API 的运行时系统设置。
- 改进首次启动配置流程。

## 0.1.0

- 初始预发布打包基线。
