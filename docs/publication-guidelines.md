# 文档公开范围

这份清单用于判断哪些文档可以放到开源仓库，哪些应该删除、脱敏或留在私有环境。

## 可以公开

以下内容可以放到开源仓库：

- 项目简介、功能边界和当前支持状态。
- 通用安装步骤，例如 SPK 安装、二进制部署、升级和卸载说明。
- 使用示例中的占位域名、占位 IP 和占位版本号，例如 `nas.example.com`、`192.0.2.10`、`<version>`。
- 飞书开放平台的通用配置步骤、公开接口地址和需要申请的通用权限名称。
- 环境变量名称、配置项含义和推荐默认值。
- 安全最佳实践，例如不要提交密钥、关闭诊断日志、管理后台仅内网开放。
- 贡献指南、发布检查清单、开发环境搭建方式。
- 高层架构说明，例如后端、前端、Helper 的职责边界。

## 不应该公开

以下内容不要放到开源仓库：

- 真实 `.env`、数据库、日志、备份、SPK 安装现场记录。
- 飞书 `App Secret`、OAuth token、DSM SID、Cookie、Synology token、临时密码。
- `/etc/shadow` 原文、片段、哈希或可还原账号状态的信息。
- Helper HMAC 密钥、生产 TLS 私钥、管理员密码或密码哈希。
- 真实企业名称、用户姓名、手机号、邮箱、部门结构、群组清单。
- 真实内网 IP、公网 IP、域名、端口映射、反向代理规则和运维拓扑。
- 未发布路线图、内部设计稿、商业计划、客户需求、未确认承诺。
- 过细的高风险实现细节、攻击复现步骤、绕过或破坏性操作说明。
- 未脱敏截图、终端输出、浏览器报错页、飞书后台页面。

## 需要脱敏后才能公开

这些内容可以公开，但必须先脱敏：

- 日志：保留错误类型和时间范围，删除 token、cookie、SID、用户名、IP、域名和请求体。
- 截图：遮盖组织名称、应用 ID、密钥、回调地址、用户列表和审计记录。
- 配置示例：使用 `nas.example.com`、`admin@example.com`、`<APP_ID>`、`<APP_SECRET>` 这类占位值。
- 排障记录：只保留通用原因和处理步骤，不保留真实环境路径之外的敏感值。

## 当前仓库保留的文档类型

当前公开仓库只保留以下文档类型：

- `README.md`：项目说明和快速入口。
- `SECURITY.md`：安全策略和敏感数据规则。
- `CONTRIBUTING.md`：贡献指南。
- `docs/spk-feishu-setup.md`：从 SPK 安装到飞书配置的用户指南。
- `docs/dsm-spk-package.md`：SPK 打包、安装、升级、卸载和排障。
- `docs/go-dsm-binary-deployment.md`：不使用 SPK 时的二进制部署方式。
- `docs/go-version.md`：Go 主线结构和本地开发方式。
- `docs/provider-development.md`：新增身份源的开发边界。
- `docs/testing.md`：测试入口和测试范围。
- `docs/release.md`：发布前检查清单。

内部设计稿和敏感机制细节不放入公开仓库。
